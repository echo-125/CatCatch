# CatCatch 猫抓

M3U8 视频流下载器，从浏览器嗅探到本地下载转码，一站式完成。

## 功能特性

### M3U8 播放列表解析

完整的 M3U8 播放列表解析引擎，支持从简单到复杂的各种 HLS 流。

**解析流程：**

```
输入 URL
  │
  ▼
HTTP GET 获取 M3U8 文本
  │  ├─ 编码自动探测: UTF-8 → Big5 → GBK → GB2312 → apparent_encoding
  │  └─ 验证 #EXTM3U 标记
  ▼
判断是否为 Master Playlist（包含 #EXT-X-STREAM-INF）
  │
  ├─ 是 → 解析所有子流，按 BANDWIDTH 降序排列，选择最高码率
  │       递归解析（最大 5 层），相对路径自动转绝对路径
  │
  └─ 否 → 解析 Media Playlist
          ├─ 逐行解析 #EXTINF 获取分片时长
          ├─ 解析 #EXT-X-KEY 获取加密信息（METHOD / URI / IV）
          ├─ 解析 #EXT-X-MEDIA-SEQUENCE 获取起始序号
          ├─ 相对路径分片 URL 通过 URI.resolve() 转绝对路径
          └─ 生成 Segment 列表（含加密元数据）
```

**关键细节：**
- 编码探测顺序考虑了中文字幕流常见的 Big5/GBK 编码
- `#EXT-X-KEY` 标签可出现在任意位置，影响其后的所有分片直到下一个 KEY 标签
- 不支持 `SAMPLE-AES` 加密（样本级加密需专用解密器），会明确报错
- 如果 M3U8 未指定 IV，自动使用媒体序列号的 16 字节大端序作为 IV（符合 HLS 规范）

### 并发下载与断点续传

基于 Kotlin 协程的高并发分片下载引擎，支持断点续传和智能重试。

**下载流程：**

```
获取分片列表
  │
  ▼
检查断点续传（已存在且非空的分片直接跳过）
  │
  ▼
协程并发下载（Semaphore 控制，默认 8 并发）
  │  ├─ 每个分片: downloadWithRetry（最多 3 次重试，指数退避 1s/2s/4s）
  │  │   ├─ 快速路径: 非空文件直接跳过（避免 CDN 限流）
  │  │   ├─ HTTP GET 下载，8KB 缓冲区流式写入
  │  │   ├─ 下载后校验文件大小（非加密分片）
  │  │   └─ 协程取消检查: ensureActive() 随时可中断
  │  │
  │  └─ 进度回调: 500ms 节流更新 UI，每 10% 输出里程碑日志
  │
  ▼
检查失败分片 → 批量重试（最多 3 轮，间隔递增 2s/4s/6s）
  │  ├─ 检测到 HTTP 401 → 自动刷新 M3U8 获取新鉴权 URL
  │  │   ├─ 重新解析 M3U8 播放列表
  │  │   ├─ 对比新旧 URL，确认是否真正更新
  │  │   └─ 用新 URL 重试失败分片
  │  │
  │  └─ 仍有失败 → 标记任务 FAILED，报告失败分片编号
  │
  ▼
全部分片就绪
```

**并发模型：**
- **任务级**：最多 3 个下载任务同时运行（可配置 1-5），超出排队
- **分片级**：每个任务内协程并发下载 TS 分片（可配置 4/8/16/32）
- **转码**：下载合并后立即释放任务槽位，转码作为独立协程后台运行

**断点续传策略：**
- 分片文件名为 `segment_0.ts`, `segment_1.ts`, ...
- 存在且大小 > 0 的分片直接跳过，不发 HTTP 请求
- 任务重试时自动清除旧的视频元信息，避免残留

### 加密视频解密

支持 AES-128-CBC 加密的 HLS 流，自动解密透明处理。

**解密流程：**

```
分片标记为加密（isEncrypted = true）
  │
  ▼
获取密钥（带缓存）
  │  ├─ ConcurrentHashMap<keyUri, keyBytes> 线程安全缓存
  │  ├─ 缓存命中 → 直接返回
  │  └─ 缓存未命中 → HTTP GET 下载密钥（复用任务 headers）
  │      └─ 验证密钥长度 = 16 字节
  ▼
CipherInputStream 流式解密
  │  ├─ 算法: AES/CBC/PKCS5Padding
  │  ├─ 密钥: 从 M3U8 #EXT-X-KEY URI 下载
  │  ├─ IV: M3U8 指定值 或 媒体序列号大端序
  │  └─ 边下载边解密，内存占用恒定 8KB
  ▼
解密后文件写入磁盘
  └─ 验证解密后文件非空
```

**设计要点：**
- 使用 `CipherInputStream` 实现流式解密，不需要将整个分片加载到内存
- 密钥缓存跨分片共享，同一加密流只下载一次密钥
- 多协程并发访问密钥缓存安全（ConcurrentHashMap）

### 分片合并与转码

下载完成后，分片合并为完整 TS 文件，再转码为 MP4。

**合并转码流程：**

```
全部分片下载完成
  │
  ▼
按序号二进制合并所有 segment_*.ts → merged.ts
  │  └─ 256KB 缓冲区，顺序写入
  │
  ▼
清理临时分片文件
  │
  ▼
提取视频元信息（分辨率、时长）
  │  └─ MediaExtractor 解析，失败不阻塞主流程
  │
  ▼
释放任务槽位，启动后台转码（不阻塞后续任务）
  │
  ▼
TS → MP4 转码（三种后端可选）
  │
  ├─ FFmpeg-kit（mode=1, 推荐）
  │   └─ ffmpeg -c copy -movflags +faststart
  │      流复制，不重新编码，速度快（~0.6s/125MB）
  │
  ├─ 系统原生（mode=2）
  │   └─ 三级降级管线:
  │      1. MediaExtractor + MediaMuxer
  │      2. 复制到 getExternalFilesDir 再试
  │      3. 纯手写 TS→MP4 muxer（PTS/DTS 重写）
  │         └─ 扫描原始字节提取 H.264 NAL
  │            手动构建 ftyp + moov + mdat MP4 容器
  │            从 TS 包解析 PTS 时间戳重写连续时间线
  │
  └─ 自动模式（mode=0）
      └─ 系统原生优先 → FFmpeg-kit 降级
  │
  ▼
SAF 复制到用户选择的目录 + 清理临时文件
  │
  ▼
发送下载完成通知（含文件路径）
```

**转码方案对比（125MB 测试文件）：**

| 指标 | FFmpeg-kit (mode=1) | 系统原生 (mode=2) | 自动 (mode=0) |
|------|---------------------|-------------------|---------------|
| 耗时 | **0.6s** | 4.9s | 5.1s |
| 输出大小 | 121,182,582 B | 121,075,956 B | 121,075,956 B |
| 时长精度 | ±0.019% | **±0.009%** | ±0.009% |
| 兼容性 | 最佳 | 依赖系统 MediaExtractor | 系统原生优先 |

**手写 muxer 技术亮点：**
- 纯 Java/Kotlin 实现，不依赖系统 mediaserver
- 从 TS 包解析 H.264 NAL unit，提取 SPS/PPS
- 解析 SPS 获取视频分辨率（支持高 profile 和帧裁剪）
- 从 PES 头部提取 PTS 时间戳，构建可变帧时长 stts 表
- H.264 SPS/PPS 自动注入（csd-0/csd-1 无效时从 TS 提取，TS 包解析 + 原始字节扫描双保险）

### 多任务队列管理

前台服务管理下载队列，支持任务的完整生命周期控制。

**任务状态流转：**

```
PENDING（排队等待）
  │
  ▼
DOWNLOADING（下载中）── 可取消 ──→ CANCELLED
  │                                   │
  │ 失败                              │ 可重试
  ▼                                   │
FAILED ←──────────────────────────────┘
  │
  │ 全部分片完成
  ▼
MERGING（合并中）
  │
  ▼
COMPLETED ──→ TRANSCODING（转码中，后台进行）
                │
                ├─ 成功 → 更新为 COMPLETED（含分辨率/文件大小）
                └─ 失败 → 保留 TS 文件，标记 FAILED
```

**队列管理：**
- `LinkedHashSet` 任务队列 + `ConcurrentHashMap<Long, Job>` 活跃任务
- `Mutex` 保护调度器，避免并发竞态
- 任务完成/取消后自动从队列取下一个排队任务
- 前台服务通知显示当前下载/转码数量
- 所有任务和转码完成后自动停止服务

**批量操作：**
- 多选模式：长按进入，底部批量操作栏滑入
- 全选/取消全选
- 批量开始（仅限 FAILED/CANCELLED/PENDING 状态）
- 批量删除

### 浏览器插件联动

通过 URL Scheme 唤起 App，支持多种触发方式，实现浏览器嗅探视频后一键下载。

**Deep Link 处理流程：**

```
触发方式
  ├─ URL Scheme: catcatch://add?url=...&title=...&headers={...}
  ├─ 浏览器分享: ACTION_SEND Intent（text/plain）
  └─ Deep Link: HTTP/HTTPS 链接在浏览器中打开
  │
  ▼
MainActivity.handleDeepLink() 解析 Intent
  │  └─ 仅允许 HTTP/HTTPS 协议，headers 过滤常用头
  │
  ▼
构建 DeepLinkData，设置 CatCatchApp.pendingDeepLink
  │
  ▼
HomeViewModel.init 延迟 100ms 后读取 pendingDeepLink
  │  └─ 消费后清空，一次性使用
  ▼
根据全局静默模式决定行为
  │
  ├─ 静默模式开启
  │   └─ 去重检查 → 直接添加任务 → 启动下载
  │       用户看到 Snackbar 提示 "静默添加: 文件名"
  │
  └─ 静默模式关闭
      └─ 填充表单（URL + 文件名 + 请求头），等待用户确认
```

**URL Scheme 参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `url` | 是 | M3U8 播放列表 URL |
| `title` | 否 | 视频标题，自动填充文件名 |
| `headers` | 否 | JSON 请求头，如 `{"origin":"...","referer":"..."}` |

**智能粘贴：**
支持 `url|文件名|请求头` 格式的一键解析，从浏览器插件复制后直接粘贴自动填充所有字段。

## 内置浏览器

### 功能概述

内置浏览器支持在 APP 内直接浏览网页并嗅探 M3U8 链接，无需跳转外部浏览器。

### UI 布局

```
┌──────────────────────────────────────┐
│ [N] 🏠 ☆ [    URL 输入框    ✈/⟳] ← → │  ← 地址栏
├──────────────────────────────────────┤
│                                      │
│  新标签页：欢迎界面                  │
│  浏览模式：WebView                   │
│                               [📥]  │  ← 悬浮嗅探按钮
├──────────────────────────────────────┤
│       主页  窗口  下载  设置         │  ← APP 底部 tab 栏
└──────────────────────────────────────┘
```

### 核心功能

1. **多标签页管理**
   - 新建、关闭、切换标签页
   - 标签管理器（侧边栏覆盖，动画效果）
   - 标签计数显示

2. **三个状态模式**
   - `NEW_TAB`：新标签页（欢迎界面）
   - `BROWSING`：网页浏览（WebView + 地址栏）
   - `TAB_MANAGER`：标签管理器（侧边栏覆盖）

3. **地址栏设计**
   - `[N]` 标签管理按钮（带黑色边框）
   - `🏠` 主页按钮
   - `☆` 收藏按钮（五角星，点击后为黑色）
   - `[URL 输入框 + ✈/⟳]` 输入框 + 发送/刷新按钮
   - `← →` 后退/前进按钮

4. **M3U8 嗅探**
   - 两层嗅探引擎：
     - Kotlin 层：`shouldInterceptRequest()` 拦截原生网络请求
     - JS 层：注入 `sniffer.js` 拦截 XHR/fetch、DOM 监听
   - 深度扫描：扫描 HTML、script 标签、video 元素、window 对象
   - 嗅探结果面板：显示链接列表，一键添加下载任务

5. **广告拦截**
   - URL 关键词过滤（ad、ads、adv、advertisement 等）
   - `shouldOverrideUrlLoading()` 拦截广告 URL

6. **其他功能**
   - 收藏功能（五角星按钮，点击后为黑色）
   - 后退/前进导航（连接 WebView）
   - 页面标题自动获取（用于文件名）
   - 加载进度条和遮罩
